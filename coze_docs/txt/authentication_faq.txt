鉴权常见问题
迁移空间后提示鉴权失败​
从个人版或扣子专业版的工作空间迁移到团队版或企业版迁移空间后，原 API 授权将失效，访问线上应用时会鉴权失败，需要重新授权，具体操作请参见​升级企业/团队版后更新 API 授权。​
令牌有效期太短，如何避免频繁更新？​
个人访问令牌的有效期最长为一个月，建议使用 OAuth 授权方式。通过 OAuth 授权，可以实现令牌的自动刷新，无需手动更新令牌，从而有效避免频繁更新令牌的问题，具体请参见​OAuth 应用管理。​
为什么授权失败，提示权限点包含 “团队/企业” 层级？​
问题现象​
创建 OAuth 应用或个人访问令牌时，提示此应用所声明的权限点包含 “团队/企业” 层级，仅团队/企业内的超级管理员或管理员可以授权应用。​
可能原因​
团队/企业层级的权限点，仅团队或企业的超级管理员和管理员创建的访问令牌能被授权该级别的权限。扣子个人账号或团队/企业中的员工不支持授权团队/企业层级的权限点。​
解决方案​
1.
确认账号角色：确认登录扣子的账号角色为团队/企业的超级管理员或管理员，如果是扣子个人账号，请单击左下角头像，切换到团队/企业的超级管理员或管理员账号。​
​
​
​
​
​
2.
检查选择的权限点：检查你选择的权限点是否包含了团队/企业层级的权限点。如果你是团队或企业的员工，你需要重新选择权限点，仅保留工作空间级别或账号级别的权限点。​
